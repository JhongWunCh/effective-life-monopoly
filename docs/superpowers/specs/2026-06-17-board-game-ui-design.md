# 經典桌遊感棋盤 UI 設計

## 背景

目前「遊戲棋盤」功能已可運作，但視覺仍偏功能表格與投影工具。下一版要把它改成更像一張真的桌遊棋盤，讓月會現場一切到棋盤頁時，大家能直覺感受到「現在要開始玩遊戲」。

## 目標

- 棋盤整體要有經典實體桌遊感，不照抄任何商標化的大富翁元素。
- 維持現有遊戲規則、卡牌邏輯、輪數、時間耗盡、結算流程，不改玩法。
- 投影時要清楚：目前隊伍、骰子狀態、機會/命運格、玩家位置、事件卡都要一眼可讀。
- 手機與窄螢幕仍可使用，不出現非預期橫向溢出。

## 視覺方向

採用「精緻實體棋盤」風格：

- 棋盤外框像一塊放在桌面上的遊戲板，有厚紙板邊框、陰影、內框線與中心遊戲區。
- 24 個外圈格子改成棋盤 tile：每格有清楚的時段色條、事件類型徽章與翻牌感。
- 未揭露格維持問號，但要更像蓋著的任務卡；已揭露格要像翻開後露出標題。
- 機會格走綠金色系，命運格走紅黑色系，一般行動/反思格維持較安定的時段色。
- 玩家 token 要更像棋子，大小與亮度提高，當前隊伍有明顯光圈或旗標。
- 中央控制台要像遊戲主持區，骰子、目前隊伍、輪次、操作按鈕整合在同一個「桌遊控制面板」中。

## 版面調整

### 棋盤本體

- 保留既有 7x7 外圈棋盤與中心控制區。
- `.board-panel` 增加遊戲板外框、桌面感背景與更明顯陰影。
- `.board-grid` 維持正方形，但在桌機上更像置中的棋盤，而不是普通 CSS grid。
- `.board-center` 改成中心徽章區加控制台，不再讓背景字和控制台彼此搶視覺。

### 棋盤格

- `.board-space` 強化 tile 感：增加上方色帶、紙卡陰影、內框、hover/目前狀態。
- 機會/命運格使用更鮮明的徽章與邊框，讓主持人和玩家遠看也能辨識。
- `is-revealed` 狀態保留，但視覺上改成「翻開」感；未揭露問號變成卡背符號。

### 主持人控制台

- `.host-controls` 改為桌遊主持區，和中心格一致。
- 骰子保留方塊，但增加骰面質感、陰影與 rolling/moving 狀態。
- 主要操作按鈕保留現有文字與行為，但視覺層級更清楚：擲骰最突出，結算次之，重設偏危險色。

### 記分板與事件卡

- 左右記分板不重做資訊架構，只改成更像玩家狀態卡。
- 目前隊伍卡片與棋盤 token 使用一致的強調語言。
- 事件卡改成抽牌區，空狀態像牌堆，抽到事件後像翻出的卡。

## 非目標

- 不新增棋盤格、不改 24 小時規則。
- 不改骰子流程、動畫 timing、卡牌選項或隨機結果。
- 不新增音效、背景音樂或複雜 3D。
- 不改首頁、角色介紹、遊戲設定、結算頁的產品內容。

## 驗收標準

- 遊戲棋盤第一眼有實體桌遊感，而不是表格或管理後台感。
- 桌機 1440x900：棋盤、左右記分板、事件卡不重疊，核心資訊可讀。
- 手機 390x844：棋盤可水平捲動但頁面本身不因其他區塊產生額外橫向溢出。
- 擲骰、移動、揭露事件、選項套用、完成結算流程維持原本行為。
- 既有測試全數通過，並新增或更新至少一個 UI 行為測試鎖定棋盤關鍵 class/文字。
- Playwright smoke 檢查桌機與手機的棋盤頁，確認非空白、無 console error、可操作到事件卡。

## 實作範圍

預計修改：

- `src/components/Board.tsx`：只在必要時增加語意 class 或非邏輯裝飾節點。
- `src/components/HostControls.tsx`：只在必要時增加包裝元素，保留 props 與行為。
- `src/components/CardPanel.tsx`：空狀態與事件卡視覺 class 可微調。
- `src/components/Scoreboard.tsx`：玩家狀態卡視覺 class 可微調。
- `src/styles.css`：主要實作位置。
- `src/App.test.tsx`：更新或新增棋盤 UI 驗收測試。

不預計修改：

- `src/game/*` 遊戲規則。
- 卡牌資料與角色數值。
- GitHub Pages 部署設定。

## 測試計畫

- RED：先補一個棋盤視覺/行為測試，例如確認棋盤頁包含經典桌遊外框 class、機會/命運格仍顯示、中心控制台仍可擲骰。
- GREEN：完成樣式與少量 JSX class 調整。
- 驗證：
  - `npm test`
  - `npm run build`
  - Playwright 本機 smoke：桌機與手機切到遊戲棋盤，點擲骰，確認事件卡出現。
  - 部署後 production smoke：正式網址切到遊戲棋盤，確認新版棋盤載入。
